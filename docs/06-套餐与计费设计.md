## 2. 计费模型

### 2.1 订阅
- 周期:`monthly` / `yearly`(年付折扣)。`subscription.current_period_*` 控制有效期。
- 续费:到期前提醒 → 自动续费(`auto_renew` + 钱包/绑定支付)或手动。
- 升级/降级:升级即时生效并按差价补费(proration,可选);降级下周期生效。

### 2.2 配额与超额
| 配额 | 超额策略(管理员可配) |
| --- | --- |
| 流量 GB | `bill`:扣钱包按 GB 加购 / `degrade`:停加速回源直连 / `block`:超限暂停 |
| 请求数 | 同上(一般 `degrade`/告警) |
| 域名数 | `block`:达上限不能再加,提示升级 |
| 防护容量 | 超过承诺容量时尽力防护 + 告警 + 建议升级(物理上限取决于基础设施) |

- 流量包加购:`order.type = addon_traffic`,即时增加本周期可用流量(写入 `quota_override` 或单独额度表)。

### 2.3 支付与钱包
- 渠道:`stripe`(国际卡)/ `wallet`(余额)/ `bank_transfer`(对公,Enterprise)。
- 钱包:充值入账 `wallet_txn(in, recharge)`;下单/超额扣费 `wallet_txn(out, order/overage)`;
  退款 `wallet_txn(in, refund)`。余额冗余在 `wallet.balance`,以流水为准对账。
- Stripe:Checkout + Webhook 验签,成功后开通/变更订阅,幂等处理重复回调。

### 2.4 试用与优惠
- 新租户可选 7 天 Starter 试用(`subscription.status=trialing`),到期转付费或降级冻结。
- 优惠券:`percent` / `fixed`,下单时校验有效期与使用上限,记 `coupon_redemption`。

## 3. 状态机

```
subscription: trialing → active → past_due(扣费失败) → active(补费) / canceled / expired
order:        pending → paid → (refunded)
                     └→ canceled
domain:       pending → verifying → (review) → active ⇄ paused → blocked
```

---

## 4. 关键计费规则(落地校验点)

1. 下单前校验:目标套餐是否存在/上架、优惠券有效、金额服务端重算(不信前端价)。
2. 支付成功(webhook/wallet)→ 事务内:写 `payment`、更新 `order.status=paid`、变更 `subscription`、
   产生 `wallet_txn`、写 `audit_log`、发通知。全部幂等。
3. 配额变更后触发受影响域名的配置重编译(如开通 WAF/CC)。
4. 周期任务:到期检查、自动续费、超额结算、试用到期降级、证书到期提醒。
