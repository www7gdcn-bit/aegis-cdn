# AegisCDN 安全引擎架构(CC + 智能 WAF + AI 风控 + Bot + 调度)

版本 v1.0 · 系统核心 · 对标 Cloudflare / Akamai / Imperva 的 **L7 智能防护**

> 已确认技术取向:**ClickHouse + Redis 日志/风控存储**、**普通 VPS 用户态 OpenResty(无 eBPF/Anycast)**、
> **实时启发式 + 多维指纹 + 风险评分 0-100 + 异常检测,离线 ML 可选**。
> 诚实分层见 README;本文聚焦"真实可落地"的 L7 引擎。

## 1. 总体数据流

```
                    ┌──────────────────────── 边缘节点(OpenResty + LuaJIT)────────────────────────┐
 client ──TLS──▶    │  ssl_client_hello_by_lua → 采集 JA3 指纹                                       │
                    │  access_by_lua → 【防护流水线 Pipeline】                                        │
                    │     0 白名单 → 1 封禁/全局拦截 → 2 WAF → 3 Bot 识别 → 4 多维限频               │
                    │     → 5 风险评分(0-100)→ 6 决策(放行/挑战/拦截)→ 7 挑战校验                │
                    │  proxy_pass → 源站                                                              │
                    │  log_by_lua → 结构化 JSON 决策日志                                              │
                    └───────▲───────────────────────────────────┬───────────────────────────────────┘
              共享状态/规则 │ Redis(计数/封禁/指纹/规则热更新)   │ 日志(本地 JSON)
                    ┌───────┴───────┐                   ┌────────▼────────────┐
                    │     Redis      │                   │   edge-agent(tail) │──批量──▶ ClickHouse
                    │  (low-latency) │                   └─────────────────────┘            │
                    └───────▲────────┘                                                       │
            规则/配置下发    │                          控制面查询/可视化                     │
                    ┌───────┴────────────────── 控制面(NestJS)────────────────────────────┴──┐
                    │  规则编译/下发 · 配额/计费 · 攻击可视化 API · 离线异常分析(可选 ML)         │
                    │  PostgreSQL(业务/配置)  +  ClickHouse(日志分析)  +  Redis(下发通道)      │
                    └────────────────────────────────────────────────────────────────────────────┘
```

**关键设计**:防护判定**必须内联在边缘 Lua**(微秒级,不能回控制面问)。控制面只负责
**规则编译下发**(写 Redis,边缘热加载)与**日志分析/可视化**(读 ClickHouse)。

## 2. 请求处理流水线(access 阶段)

| 步 | 模块 | 动作 | 命中后 |
| --- | --- | --- | --- |
| 0 | `whitelist` | IP/ASN/UA 白名单 | 直接放行(跳过后续) |
| 1 | `ban` | Redis 封禁名单 + 全局黑名单 | 403 / 444 |
| 2 | `waf` | OWASP 规则集 + 自定义规则匹配 | block / challenge / log |
| 3 | `bot` | 自动化/伪 Bot 识别,信誉评分 | 标记 bot_score,喂给风控 |
| 4 | `ratelimit` | 多维度多窗口限频 | 超限 → 加分 / 挑战 / 封禁 |
| 5 | `risk` | 多特征加权 → Risk Score 0-100 | 写 ngx.ctx |
| 6 | 决策 | 阈值映射 | ≥block→拦截;≥challenge→挑战;否则放行 |
| 7 | `challenge` | 校验挑战 Cookie(HMAC) | 通过放行;否则下发挑战页 |
| — | `log` | log_by_lua 写决策 JSON | → agent → ClickHouse |

阈值与开关按域名存 Redis(`aegis:cfg:<domain>`),控制面下发,边缘 `lua_shared_dict` 缓存 + 定时刷新。

## 3. 超强 CC 防护(真实实现)

### 3.1 多维度限频
维度 key 统一格式 `aegis:rl:<domain>:<dim>:<value>:<window>`,维度:
`ip` / `uri` / `cookie` / `session` / `ua` / `asn` / `country`。
窗口:`10s` / `1m` / `5m` / **动态窗口**(攻击模式下自动收紧阈值)。

### 3.2 三种算法(`ratelimit.lua`,可按域名/维度选择)
- **固定/滑动窗口(Sliding Window)**:Redis 双窗口加权计数(当前窗口 + 上一窗口按时间占比加权),
  避免固定窗口边界突刺。用 `INCR`+`EXPIRE`,O(1)。
- **令牌桶(Token Bucket)**:Redis Lua 原子脚本维护 `tokens/last_refill`,支持突发 `burst`。
- **漏桶(Leaky Bucket)**:Redis Lua 脚本恒定漏出速率,平滑流量。

> 三种算法均以 **Redis Lua 脚本保证原子性**(避免并发竞态),脚本 `SCRIPT LOAD` 后用 `EVALSHA`。

### 3.3 智能封禁(`ban.lua`)
- 临时封禁:`SET aegis:ban:<ip> 1 EX <ttl>`(自动恢复)。
- 永久封禁 / CIDR / ASN / 国家:存 Redis set + 全局黑名单表(控制面下发)。
- **自动学习**:同一 IP/ASN 在窗口内多次触发挑战失败或高风险 → 升级封禁时长(指数退避)。
- 白名单优先于一切封禁。

## 4. AI 行为识别与指纹(真实 + 启发式)

### 4.1 指纹采集
| 指纹 | 采集方式 | 模块 |
| --- | --- | --- |
| **JA3 / TLS** | `ssl_client_hello_by_lua` 解析 ClientHello(cipher/ext/曲线)→ MD5 | `fingerprint.lua` |
| **UA / HTTP 指纹** | 头部顺序、缺失头、HTTP 版本、Accept 特征 | `fingerprint.lua` |
| **Canvas / WebGL / 浏览器指纹** | 挑战页注入 JS 采集 → 回传 → 存 Redis 关联 | `challenge.lua` + 前端 collector |

### 4.2 自动化/Bot 识别(`bot.lua`)
- **签名库**:curl、python-requests、Go-http-client、Java、Scrapy、PhantomJS、HeadlessChrome、Selenium/Puppeteer 痕迹。
- **缺失浏览器特征**:无 `Accept-Language`、异常 `Accept`、无 Cookie 能力、TLS 与 UA 不一致(JA3 与声称浏览器矛盾)。
- **HTTP Flood**:高频 + 单一指纹 + 无静态资源加载行为。
- **伪 Google/Bing Bot**:声称 GoogleBot → 反查 rDNS(PTR)+ 正查确认归属 `googlebot.com`,不符即标记 fake。

### 4.3 风险评分(`risk.lua`,0-100)
加权求和(权重可调,存 Redis):
```
score = w1*rate_pressure + w2*bot_signal + w3*fp_anomaly
      + w4*geo_risk + w5*history_penalty + w6*waf_soft_hit
```
- `rate_pressure`:超出阈值的比例;`bot_signal`:自动化命中强度;`fp_anomaly`:指纹矛盾度;
- `history_penalty`:该 IP/ASN 近期违规累计;`geo_risk`:高风险地区;`waf_soft_hit`:WAF 观察模式命中。
- 输出分桶:0-30 放行 / 30-70 挑战 / 70-100 拦截(阈值按域名 + 攻击模式动态)。

### 4.4 离线 ML(可选增强,本期埋点不强依赖)
- ClickHouse 攻击/访问明细 → 离线训练**异常检测**(Isolation Forest / 频次基线 3σ)→ 产出
  **可疑指纹/ASN 评分表** 回灌 Redis,作为 `history_penalty` 输入。
- **0day/未知攻击**:走**行为基线偏离 + 异常检测**给出"疑似"评分并**建议规则**,由人工审核上线,
  **不宣称"AI 自动挡 0day"**(诚实)。

## 5. 企业级 WAF(`waf.lua` + 规则引擎)

### 5.1 内置规则集(OWASP Top 10 方向)
SQLi、XSS、RCE/命令执行、文件上传、SSRF、XXE、路径穿越、CSRF 校验、WebShell 特征、协议异常。
每条规则:`{id, name, target, operator, pattern, action, severity, phase}`。
`target` ∈ uri/args/body/headers/ua/cookie/referer;`operator` ∈ regex/contains/eq/cidr/in;
匹配用 `ngx.re.find(..., "jo")`(JIT + 编译缓存)。

### 5.2 规则形态与热更新
- **运行时**:规则编译为 **JSON** 存 Redis(`aegis:waf:<domain>`),边缘 `lua_shared_dict` 缓存 + 版本号,
  控制面改动 → bump 版本 → 边缘下个请求或定时器热加载,**无需 reload**。
- **作者侧**:支持 **YAML / 条件表达式** 编写,由控制面编译成运行时 JSON(YAML 不在边缘解析,边缘只吃 JSON,快)。
- **条件表达式示例**(控制面 DSL → 编译):
  ```yaml
  - name: wp-login-bruteforce
    when:
      all:
        - uri contains "/wp-login"
        - rate(ip, 1m) > 30
    then: challenge
  ```
- **AI 风险规则**:`risk_score > N` 作为规则条件之一,把评分接进规则引擎。

### 5.3 动作
`allow` / `log`(观察模式,仅记录喂风控) / `challenge`(转挑战) / `block`(403/444)。

## 6. 动态挑战系统(`challenge.lua`)

| 类型 | 实现 |
| --- | --- |
| **JS Challenge / 5 秒盾** | 返回挑战页:JS 计算 token、采集浏览器指纹,种 Cookie `aegis_clr` 后自动跳回。token = `HMAC(secret, ip|ua|window)`,边缘校验。 |
| **滑块 / 验证码 / 人机** | 接入第三方(hCaptcha/Turnstile 风格)或自建,验证通过签发通行 Cookie。 |
| **触发条件** | 自动触发 / 按风险评分 / 按国家 / 按频率,任意组合。 |

通行 Cookie 有效期 + 单 IP 通行态缓存于 Redis,避免重复挑战。

## 7. 全球节点调度(普通 VPS 版,无 Anycast)

- **DNS 调度**:客户 CNAME → 平台权威 DNS,按 GeoDNS + 健康度返回最优节点 IP(无 Anycast 时的等效方案)。
- **健康检测**:`edge-agent` 心跳 + 控制面主动探测;不健康节点自动从 DNS 解析池摘除(自动切换)。
- **动态路由**:按节点负载/RTT/容量加权;专属节点(Enterprise)固定解析到独享组。
- > BGP/Anycast/eBPF 在普通 VPS 不可得,文档不承诺;后续若上裸金属或云清洗可平滑扩展。

---

## 8. 攻击可视化(ClickHouse 驱动)

- 边缘 JSON 决策日志 → `edge-agent` 批量写 ClickHouse(`attack_log` / `request_log` 宽表,按天分区,TTL 按套餐)。
- 控制面聚合 API(物化视图 / 预聚合)供后台:实时攻击地图、QPS 曲线、攻击来源国家 / Top IP / Top ASN、
  攻击类型分布、风险评分趋势。前端 Apple × Cloudflare 风格(ECharts)。
- 实时性:近实时(秒~十秒级,批量入库);需要更实时可加 Redis Stream 热路径。
