# AegisCDN 产品需求文档 (PRD)

版本 v1.0 · 2026-05-21

---

## 1. 产品定位

AegisCDN 是面向**企业客户**的商用级 **高防 CDN / DDoS 防护 / 网站加速 / WAF 安全防护 / 边缘节点加速** SaaS 平台。
客户通过 **CNAME 接入**,即可获得内容加速、缓存、HTTPS、CC/WAF 防护、IP/地区拦截等能力,并在控制台实时查看
流量、攻击、防护与计费数据。

- **目标客户**:有公网业务、面临 CC/爬虫/Web 攻击、需要加速与防护的中大型网站、电商、游戏、API 服务商。
- **核心价值**:一键接入、智能防护、全球加速、永远在线(Always Online)、可视化运营。
- **风格定位**:Apple 官网式高级简洁 + Cloudflare 式专业可信。

### 1.1 商业模式

SaaS 订阅制:按套餐(Starter / Business / Enterprise)售卖防护容量、流量额度、域名数与功能项;
支持购买 / 续费 / 升级 / 流量包加购 / 余额钱包。Enterprise 走定制报价 + 专属节点。

## 2. 角色与权限 (RBAC)

| 角色 | 说明 | 主要权限域 |
| --- | --- | --- |
| **访客 (guest)** | 未登录 | 浏览官网、注册、查看套餐 |
| **客户 (user)** | 企业客户(租户) | 自己的域名、配置、统计、订单、钱包 |
| **客户子账号 (member)** | 同租户下子用户 | 受租户管理员授权的子集(只读/运维) |
| **运营 (operator)** | 平台运营 | 用户/订单/工单/公告,只读节点 |
| **管理员 (admin)** | 平台超管 | 全部:用户、套餐、节点、防护策略、封禁、系统日志 |

> 采用 **租户(tenant)隔离**:每个客户是一个 tenant,域名/配置/日志/订单按 `tenant_id` 隔离;
> 子账号属于同一 tenant。平台角色(operator/admin)跨租户。权限用 `role + permission` 组合,RBAC 表见 DB 设计。

## 3. 关键业务流程

### 3.1 客户接入流程(CNAME)
1. 注册 / 登录 → 实名/企业资料(可选)→ 选购套餐(或先试用)
2. 控制台「添加域名」→ 填写源站(回源地址)、协议、端口
3. 平台分配接入 CNAME(如 `d-8f3a.aegis-cdn.net`)与校验 TXT
4. 客户在自己 DNS 处把域名 CNAME 到该值
5. 平台校验解析生效 → 自动签发 HTTPS 证书(ACME)→ 编译边缘配置并下发
6. 域名状态 `pending → verifying → active`,接入完成,流量开始经过边缘

### 3.2 配置下发流程(控制面 → 数据面)
1. 客户在控制台修改缓存/WAF/CC/ACL/回源等配置 → 写入 DB,产生 `config_version`
2. API 的 **配置编译器** 把该域名配置渲染成 OpenResty server/location + Lua 数据
3. 通过队列下发任务;边缘 `edge-agent` 拉取最新版本 → 原子写入 → `nginx -t` → reload
4. 下发结果回报,控制台显示「已生效 / 灰度中 / 失败回滚」

### 3.3 日志与统计流程(数据面 → 控制面)
1. OpenResty 按请求写访问日志 + 攻击事件(被 WAF/CC/ACL 拦截)
2. `edge-agent` 批量压缩上报到日志接收端(或对象存储 + 队列)
3. BullMQ worker 聚合成分钟/小时/天粒度:流量、请求数、状态码、命中率、攻击次数、Top 攻击 IP/地区
4. 控制台与管理后台读取聚合表渲染图表;原始攻击日志保留可查询期(按套餐)

### 3.4 计费流程
- 订阅周期(月/年)+ 配额(流量 GB、防护容量 G、域名数、请求数)
- 超额:阻断 / 降级 / 计费加购(可配置策略),余额钱包扣费
- 续费/升级:订单 → 支付(Stripe / 余额 / 对公)→ 开通/变更配额 → 配置生效

## 4. 功能清单

### 4.1 官网前台(Marketing)
- Hero 主视觉、产品优势、全球节点地图、防护能力展示、套餐价格、客户案例、FAQ、联系/立即接入 CTA
- 注册 / 登录 / 找回密码;响应式;SEO(meta/OG/JSON-LD/sitemap)

### 4.2 用户控制台(Console)
- **概览仪表盘**:实时流量、请求数、带宽、命中率、防护次数、攻击趋势、域名健康
- **域名管理**:添加/删除域名、CNAME 接入与校验、状态、批量
- **加速与缓存**:缓存规则(按路径/后缀/状态码/TTL)、强制缓存、缓存清除(刷新/预热)
- **回源配置**:回源地址(多源/负载/主备)、回源协议、回源 Host、超时、回源鉴权
- **HTTPS**:上传证书 / ACME 自动签发 / 强制 HTTPS / HSTS / TLS 版本
- **安全防护**:CC 防护(阈值/模式/人机校验)、WAF(规则集开关/自定义规则/拦截动作)、
  IP 黑白名单、国家/地区拦截、User-Agent / Referer 拦截、Bot 识别、限速
- **统计与日志**:流量统计、攻击日志、防护次数、带宽、请求数、命中率、状态码分布、Top N
- **套餐与计费**:当前套餐与配额、购买/续费/升级、流量包、订单、钱包余额与流水
- **账号**:资料、子账号与权限、API 密钥、通知设置、工单

### 4.3 管理员后台(Admin)
- 用户管理(查/封禁/改套餐/重置)、套餐管理(增删改/上下架)、域名管理与**接入审核**
- 全量客户流量与攻击事件、节点管理(增删/健康/容量/分组/灰度)、防护策略模板
- 订单管理、余额/钱包管理、退款、公告管理、工单处理
- **手动封禁域名 / IP**、全局黑白名单、系统日志与审计、运营数据看板(收入/活跃/防护总量)

## 5. 非功能需求

| 维度 | 要求 |

| 性能 | 边缘 P95 转发延迟可量测;控制台接口 P95 < 300ms;统计走预聚合 |
| 可用性 | 控制面高可用;边缘多节点,单节点故障不影响其余;配置下发失败自动回滚 |

| 可扩展 | 控制面/数据面分离;节点水平扩展;日志走队列削峰;配置版本化可灰度可回滚 |
| 可观测 | 节点健康检查、配置下发可追踪、计费可对账、操作可审计 |
| 合规 | 见 §6 |
| 国际化 | 文案以中文为主、Hero 含英文;预留 i18n(中/英) |

## 6. 合规与边界(重要)

- **真实防护边界**:平台在 L7 层(CC/WAF/限速/ACL/缓存/HTTPS)提供真实能力;**L3/L4 容量级 DDoS 清洗
  取决于上游带宽 / Anycast / 云清洗**,套餐"防护容量"为售卖承诺,需基础设施支撑,文档与销售页不夸大为"软件即带宽"。
- **滥用治理**:高防 CDN 易被用于隐藏恶意站点(钓鱼/赌博/诈骗)。平台需具备:接入审核、违规域名封禁、
  滥用举报入口、KYC(企业资料)、敏感行业拒绝接入。本平台仅服务**合法合规业务**。
- **数据**:攻击日志含访客 IP,按最小化与保留期处理;遵守客户所在地法律。
- 与 [[project-swiftvpn]] 等既往项目相同的合规审慎态度:对违法用途明确拒绝。
