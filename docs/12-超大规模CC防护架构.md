# AegisCDN 超大规模 Layer7 CC 防护架构

版本 v1.0 · 对标 Cloudflare / Akamai / 腾讯云高防 / 阿里云高防 的 **L7 智能 CC 防护**

> 边界声明(诚实):本文设计的是 **L7(HTTP)层** 的企业级分布式 CC 防护,可真实落地在
> OpenResty+LuaJIT+Redis 上。**eBPF/XDP、Anycast、L3/L4 容量级清洗**属基础设施层,需裸金属/内核权限/
> 网络运营能力或叠加上游云清洗 —— 本文给出对接位置但不假称软件能凭空产生带宽。
> 单节点 100万 QPS 是**设计目标**:靠"本地内存快路径 + Redis 分片 + 内核调优 + 多数请求本地裁决"逼近,
> 实际峰值取决于硬件;架构按此目标设计。

---

## 0. 设计总纲:三层裁决,逐层降级

CC 防护的关键矛盾:**既要微秒级裁决(否则自己成瓶颈),又要全网一致(攻击者切 IP 仍被识别)**。
解法是 **三层(L0/L1/L2)裁决**,绝大多数请求在 L0 本地内存命中,不触网络:

```
请求 ──▶ L0 本节点 lua_shared_dict(微秒级,无网络)
            │  命中本地黑名单/超阈值 → 直接处置
            │  Bloom/Cuckoo 过滤器:绝大多数"正常"请求 O(1) 放行
            ▼ (仅可疑请求下沉)
         L1 Redis 集群(毫秒级,跨节点精确计数 + 全局信誉)
            │  分布式限流真值、全局信誉表(攻击者切 IP 仍命中指纹信誉)
            │  pub/sub 即时广播封禁 → 各节点回填 L0
            ▼ (异步,不在请求路径)
         L2 Kafka → 流式分析(攻击模式学习 / 自动生成规则)→ 回写 Redis
```

**90%+ 的请求在 L0 完成裁决**(本地命中信誉/未触阈值的 Bloom 快放行),只有"可疑"流量下沉 L1,
攻击数据异步进 L2。这是 100万 QPS/节点 可行的根本:**热路径几乎不打 Redis**。

## 1. 分布式 CC 防护架构(多节点 / 跨节点识别)

### 1.1 实体(Entity)— 不只看 IP
攻击者轮换 IP 是常态,**单看 IP 必失效**。本架构以**复合实体指纹**为识别主键:
```
entity = hash(JA3 + HTTP指纹 + ASN [+ 可选 Cookie/Session])
```
IP 只是维度之一。攻击者换 IP 但 TLS 栈/HTTP 库/ASN 不变 → JA3/HTTP 指纹稳定 → **同一 entity,全网仍识别**。

### 1.2 全局信誉表(跨节点同步)
- 任一节点判定某 entity/IP 为攻击 → 写 **Redis 全局信誉** `aegis:rep:<entity>`(带 TTL + 风险分),
  并 `PUBLISH aegis:rep:flag <entity>`。
- 所有节点订阅该频道(privileged timer 里 `lua-resty-redis` subscribe),收到即写入**本地 shared_dict**,
  后续该 entity 在**任何节点**首个请求即被 L0 命中。**这就是"切 IP 也跑不掉"**。
- 冷启动/订阅丢失兜底:L0 对"中等可疑"的 entity 做一次 Redis `GET aegis:rep:<entity>` 校准。

### 1.3 数据通道
| 通道 | 用途 | 时延 |
| --- | --- | --- |
| **lua_shared_dict** | 本节点热计数 / 本地信誉 / Bloom 过滤器 | 微秒,无网络 |
| **Redis 集群** | 跨节点精确限流真值 + 全局信誉 | 毫秒,分片扩展 |
| **Redis pub/sub** | 封禁/信誉即时广播到全网 | 亚秒 |
| **Kafka** | 决策日志洪流 → 流式分析 / 自动规则 | 异步,削峰 |

> Redis 集群按 key 分片(限流 key 自带 entity/维度,天然可分片);信誉表可用单独 Redis 或同集群。

## 2. 超高性能风控(内存级 / 微秒级 / 不落盘)

- **内联 OpenResty/LuaJIT**:`access_by_lua` 阶段裁决,全部在内存;不落磁盘、不阻塞、不回控制面问。
- **Bloom / Cuckoo 过滤器**(shared_dict 实现):热路径第一道。
  - Bloom:判"**一定不在**可疑集 → 立即放行"(无误放,可能误判为可疑→下沉,代价小)。
  - Cuckoo:支持删除(信誉过期可移除),适合动态可疑集。
  - 效果:海量正常请求 O(1) 位运算放行,**不查 Redis、不跑复杂评分**。
- **Redis 优化**:连接池 + pipeline + `EVALSHA`(脚本预载),限流/信誉脚本原子;失败 **fail-open**(CDN 可用性优先)。
- **eBPF/XDP(L3/L4,基础设施层)**:SYN flood、UDP flood、畸形包在**内核态 XDP**丢弃,userspace 之前完成。
  需 root/内核/裸金属。与 L7 互补:XDP 扛容量,L7 扛 CC。普通 VPS 不可得 → 文档标注,不强依赖。

## 3. 智能行为识别(多维,不只看 IP)

| 维度 | 采集/分析 | 信号 |
| --- | --- | --- |
| **JA3 / TLS 指纹** | ClientHello(cipher/ext/曲线)MD5 | 库/工具指纹;与声称浏览器矛盾即异常 |
| **HTTP / Header 指纹** | 头顺序、缺失头、HTTP 版本、Accept 特征 | curl/python/Go 等库特征明显 |
| **浏览器指纹** | 挑战页 JS:Canvas/WebGL/字体/时区/分辨率 | 无头浏览器特征、指纹一致性 |
| **Cookie / Session 行为** | 是否支持 Cookie、Session 连续性 | 脚本类多无 Cookie 能力 |
| **请求轨迹** | 路径序列、是否加载静态资源、访问熵 | Flood 只打单点、无资源加载 |
| **鼠标 / 人机行为** | 挑战页采集移动/点击/停留 | 自动化无人机行为 |

**自动化识别矩阵**:Selenium/Puppeteer(webdriver 痕迹+无头特征)、Curl/Python-Requests(UA+JA3+缺头)、
HTTP Flood(高频+单指纹+无资源)、API 滥刷(高频打 /api/* + 无浏览器特征)、Bot 网络(分散 IP 但同指纹/同轨迹)。

## 4. 动态风控引擎(Risk Score 0-100)

```
score = w_rate·限频压力 + w_bot·自动化信号 + w_fp·指纹矛盾
      + w_rep·全局信誉(跨节点!)+ w_traj·轨迹异常 + w_geo·地区 + w_waf·软命中
```
分数 → 动作映射(阈值按域名 + **attack-mode 动态收紧**):
```
0–30   放行
30–50  JS Challenge
50–70  验证码 / 滑块 / 5秒盾
70–90  封禁(临时)
90–100 封禁(升级)+ 写全局信誉广播全网
```
`w_rep` 让"曾在别处作恶"的 entity 一进来就高分 —— 这是跨节点防护的核心权重。

## 5. 高级限速系统(多维 × 多算法 × 两层)

- **维度**:IP / URI / Cookie / Token / Session / UA / ASN / 国家。
- **算法**:Token Bucket(突发)/ Sliding Window(平滑,消边界突刺)/ Leaky Bucket(恒速)。
- **两层计数**:
  - **L0 本地**(shared_dict):每节点近似计数,微秒级,**先挡掉单节点视角的明显超频**。
  - **L1 Redis**(EVALSHA 原子):跨节点精确真值,本地可疑时才查 —— 防"分散到多节点绕过单点限流"。
- **百万 QPS / 秒级同步**:L0 吸收绝大多数;L1 按 key 分片到 Redis 集群;阈值变更经 pub/sub 秒级下发各节点。
- **动态调整**:attack-mode 下自动调小窗口/阈值(见 §8 自动化)。

## 6. 真实攻击模式防护

| 模式 | 特征 | 处置 |
| --- | --- | --- |
| **HTTP Flood**(GET/POST 海量) | 高频 + 单/少指纹 + 无静态资源加载 | 限频 + 指纹信誉 + 挑战 → 封禁 |
| **API Flood**(刷 /api/*) | 高频打 API、无浏览器特征、Token 维度异常 | 单 Token/URI 限频 + Bot 识别 |
| **登录爆破**(/wp-login //login //api/auth) | 同路径高频、密码字典、失败率高 | URI 维度严格限频 + 强制挑战/验证码 |
| **搜索攻击** | 大量异参搜索打穿缓存 | 搜索路径限频 + 参数熵检测 + 缓存兜底 |
| **动态页攻击(绕缓存打源)** | 带随机参/no-cache 直奔动态页 | 识别 cache-buster、强制挑战、源站限速 |
| **Bot 爬虫 / AI Bot / 扫描** | 已知 UA/ASN、轨迹规律、扫敏感路径 | Bot 信誉分类 + 真假搜索引擎 rDNS 校验 |

## 7. 源站保护(防绕过 CDN 直击源站)

- **隐藏真实 IP**:源站只暴露给边缘;DNS 不解析真实源 IP。
- **回源鉴权 / 签名**:边缘回源加 `X-Aegis-Sign: HMAC(secret, ts+path)` + `X-Aegis-Token`;源站校验,拒绝无签名直连。
- **回源白名单**:源站防火墙/安全组**仅放行边缘节点 IP**(根本性防绕过)。
- **源站限速**:边缘对回源也限速,防穿透型攻击打爆源站。
- **健康保护**:源站异常时边缘返回缓存/静态兜底页(Always Online)。

## 8. 自动化防御(自学习,无需人工)

- **attack-mode 自动触发**:全局/单域名 QPS 超基线(Redis 滑动统计)→ 自动进入 attack 模式 → 自动收紧阈值、
  提高挑战比例;QPS 回落 → 自动退出。
- **自动封禁 + 指数退避 + 自动恢复**:重复触发的 entity 封禁时长指数增长(已实现 `ban.lua`),TTL 到自动恢复。
- **自动学习 / 自动生成规则(L2)**:Kafka 决策日志 → 流式/离线异常检测(频次基线 3σ / Isolation Forest)→
  识别异常指纹/ASN/URI 模式 → **生成候选规则写回 Redis**(高置信自动生效,中置信转人工)。
  诚实:对**已知模式**可全自动;对**未知 0day** 给"疑似+建议",高风险动作仍建议人审,不宣称"AI 全自动挡 0day"。

## 9. 攻击可视化(已实现 /admin/security,本架构扩展指标)

实时:全局 QPS、各节点 QPS、当前拦截量/拦截率、攻击国家/ASN/URI、Top IP、风险分趋势、实时攻击地图。
数据:边缘决策日志 → edge-agent → ClickHouse →(可前置 Kafka 削峰)→ 后台聚合。

## 10. 性能与高可用(逼近 100万 QPS/节点)

| 手段 | 说明 |
| --- | --- |
| 本地快路径 | 90%+ 请求在 L0 shared_dict + Bloom 裁决,不触 Redis |
| Redis 分片 + pipeline + EVALSHA | 下沉流量按 key 分片;脚本预载减少 RTT |
| 连接池 / keepalive | 上下游长连接,减少握手 |
| Fail-open | Redis 抖动不影响转发,可用性优先 |
| Worker / OS 调优 | `worker_processes auto`、`worker_connections`、`somaxconn`、`nofile`、TCP 调优 |
| eBPF/XDP(基础设施) | L3/L4 内核态丢包,卸载 userspace 压力 |
| 水平扩展 | 节点无状态(状态在 Redis),按地域横向加;DNS 智能调度 + 健康摘除 |

> 实测峰值取决于硬件(CPU/网卡/内存)与请求复杂度。架构保证"**绝大多数请求走本地内存**",
> 这是高 QPS 的前提;具体 100万 需在目标硬件上压测调优。
