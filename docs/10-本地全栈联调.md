# AegisCDN 本地全栈联调

把所有部分(官网/控制台 + 控制面 API + 边缘引擎 + 日志管道 + 可视化)在本地连起来跑,
端到端验证两条核心链路:
1. **控制面 → 数据面**:控制台配置防护 → 下发 Redis → 边缘 OpenResty 热加载生效。
2. **数据面 → 可视化**:边缘决策日志 → edge-agent → ClickHouse → 后台实时图表。

> 前置:**Docker(含 compose)** + **Node 18+**。Windows 建议用 WSL2 里的 Docker。
> 本机若无 Docker 无法运行(代码本身已逐段 build 通过)。

---

## 1. 起后端全链路(一条命令)

```bash
# 仓库根目录
docker compose up -d --build
```

启动:postgres、**redis(控制面/边缘共用)**、**clickhouse(agent写/api读)**、demo-origin(whoami)、
openresty(边缘引擎)、edge-agent(日志管道)、api(NestJS 控制面,启动自动 `prisma db push` 建表)。

检查:
```bash
docker compose ps
curl http://localhost:4000/api/v1/health           # {"ok":true,"db":"up"}
curl -H "Host: demo.test" http://localhost:8080/aegis-health   # ok
```

## 2. 起前端

```bash
npm install            # 首次
npm run dev:web        # http://localhost:3000(浏览器经 localhost:4000 调 API)
```

## 3. 端到端走一遍

### 3.1 注册 + 接入域名
1. 打开 http://localhost:3000 → **注册**(首个账号=平台管理员)。
2. 控制台 → **域名管理** → 添加域名 `example.com`(源站随意,demo 边缘统一回源到 whoami)。
3. 进入该域名详情 → 点 **「模拟审核通过并激活」**。
   - 这一步会把 `aegis:cfg:example.com` 写入 Redis(`enabled=true`)。

验证下发到 Redis:
```bash
docker compose exec redis redis-cli get "aegis:cfg:example.com"
```

### 3.2 验证边缘真实生效(控制面→数据面)
边缘配置缓存 TTL 10s,激活后稍等 10 秒再测。`Host` 头指定为你接入的域名:

```bash
BROWSER='Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 Chrome/124 Safari/537.36'

# 正常浏览器 → 放行 200(返回 whoami 源站回显)
curl -s -o /dev/null -w '%{http_code}\n' -H "Host: example.com" -H "User-Agent: $BROWSER" -H "Accept-Language: zh-CN" http://localhost:8080/

# SQL 注入 → WAF 拦截 403
curl -s -o /dev/null -w '%{http_code}\n' -H "Host: example.com" "http://localhost:8080/?id=1%20union%20select%20pwd%20from%20users"

# curl 默认 UA(自动化)→ 5 秒盾挑战 503
curl -s -o /dev/null -w '%{http_code}\n' -H "Host: example.com" http://localhost:8080/
```

### 3.3 改策略 → 立即生效
1. 域名详情 → **WAF** → 加一条自定义规则:`target=uri op=contains pattern=/wp-login action=block`,保存。
2. 等 ~10s,测试:
   ```bash
   curl -s -o /dev/null -w '%{http_code}\n' -H "Host: example.com" "http://localhost:8080/wp-login.php"   # 403
   ```
3. **访问控制** 加 `ip / deny / <你的测试IP或某网段>` → 该来源直接 403。
4. 详情页「查看下发 JSON」可看到实际写进 Redis 的完整配置。

### 3.4 验证可视化(数据面→后台)
1. 制造一些流量(重复上面的 curl,或跑 `cd edge && bash test.sh http://localhost:8080`,注意带 Host 头的版本)。
2. 验证日志已进 ClickHouse:
   ```bash
   curl 'http://localhost:8123/?query=SELECT count() FROM aegis.request_log'
   curl 'http://localhost:8123/?query=SELECT action,count() FROM aegis.request_log GROUP BY action FORMAT JSON'
   ```
3. 打开 http://localhost:3000/admin/security —— 右上角应显示 **「ClickHouse 实时」**(而非样本),
   KPI / 趋势 / Top IP / 攻击事件来自真实日志。

## 4. 链路全景

```
浏览器 ─ localhost:3000 (Next 控制台/官网/后台)
            │  REST (localhost:4000)
            ▼
        NestJS api ──写──▶ Redis ◀──读── OpenResty(localhost:8080)──proxy──▶ demo-origin
            │  读                              │  log_by_lua
            ▼                                  ▼
        ClickHouse ◀──批量写── edge-agent ◀─tail─ access.json.log
```

## 5. 排障

- `/admin/security` 仍显示「样本数据」:说明前端没连上 api,或 api 没连上 ClickHouse。查 `docker compose logs api`、确认 `CLICKHOUSE_URL` 指向 `http://clickhouse:8123`。
- 边缘改了策略不生效:等满 10s(配置缓存 TTL);确认 api 与 openresty 用的是**同一个 redis**(本 compose 已保证)。
- api 起不来:看 `docker compose logs api`,通常是 `prisma db push` 连不上 postgres(等 postgres 就绪后会重试,可 `docker compose restart api`)。
- 真实客户端 IP:本 demo 直连边缘;生产在边缘前有 LB 时,按 `nginx.conf` 的 `set_real_ip_from` 配置可信上游网段。

## 6. 收尾

```bash
docker compose down          # 停止
docker compose down -v       # 停止并清空数据卷(pg/日志/offset)
```
