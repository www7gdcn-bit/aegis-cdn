## 1. 技术栈

| 层 | 选型 |
| --- | --- |
| 前端 | Next.js 14 (App Router) · TypeScript · Tailwind CSS · Framer Motion · ECharts/Recharts |
| 后端 | NestJS · Prisma · PostgreSQL 14+ · Redis · BullMQ · JWT(httpOnly)· class-validator |
| 边缘 | OpenResty(Nginx + Lua)· lua-resty-limit-traffic · GeoIP2 · edge-agent(Go 或 Node) |
| 基础设施 | Docker / docker-compose · Nginx · ACME(acme.sh / lego)· 宝塔可部署 |
| 共享 | `packages/shared`(TS 类型、DTO、常量,前后端共享) |

## 3. 配置编译与下发(核心机制说明)

1. **编译**:`config` 模块把某域名的 origin/cache/cert/cc/waf/acl 聚合渲染为一个 **快照 JSON**(`config_version.snapshot`)
   + 生成 OpenResty `server{}` 片段与 Lua 数据(规则表、名单)。
2. **入队**:为目标节点组的每个节点建 `deploy_task`,投递到 BullMQ。
3. **拉取**:`edge-agent` 周期 `GET /edge/config?since=cursor` 拿增量 → 原子写 `conf.d/<domain>.conf` 与 Lua data →
   `nginx -t` 通过则 `reload`,失败则保留旧版并回报 `failed`(平台标记并可回滚)。
4. **回报**:`POST /edge/config/:taskId/ack`,平台汇总后把 `config_version.status` 置 `active`。
5. **灰度**:可先下发到部分节点,验证后全量;**回滚**=把旧版本重新入队下发。

## 4. 日志与统计管道

OpenResty `log_by_lua` 写结构化日志 → `edge-agent` 批量压缩 → `POST /edge/logs/ingest` →
入 Redis Stream/队列 → BullMQ worker 聚合写 `traffic_stat`(分钟桶,再 rollup 到小时/天)与 `attack_event`。
控制台查询只读预聚合表,保证快。

## 5. 分阶段开发计划(每阶段实测可跑)

| 阶段 | 内容 | 交付物 |
| --- | --- | --- |
| **① 骨架** | Monorepo、Next+Nest 起步、Prisma 建库、Docker compose、健康检查、CI lint | 一键起服务 |
| **② 官网** | Apple 风格设计系统 + 首页全板块 + pricing/network/docs + 响应式 + SEO | 可上线官网 |
| **③ 鉴权与租户** | 注册/登录/JWT(httpOnly)/找回/邮箱验证、RBAC、租户隔离、子账号、API Key | 账号体系 |
| **④ 域名接入** | 添加域名→分配 CNAME→TXT/解析校验→接入审核→状态机;回源/HTTPS(ACME) | 能接入真实域名 |
| **⑤ 配置编译与边缘下发** | config 编译器 + deploy + edge-agent + OpenResty 真实反代/缓存 | 流量真过边缘 |
| **⑥ 日志与统计** | 日志管道 + 聚合 + 控制台仪表盘/统计图表 | 真实数据可视化 |
| **⑦ 安全功能** | CC(limit-req)、WAF(Lua 规则集 + 自定义)、ACL(IP/地区/UA/Bot)、攻击日志 | 防护真实生效 |
| **⑧ 套餐与计费** | plan/subscription/order/payment(Stripe+钱包)/配额门控/超额策略/续费任务 | 可售卖 |
| **⑨ 管理后台** | 用户/审核/套餐/订单/节点/策略/封禁/公告/工单/审计/系统设置 | 可运营 |
| **⑩ 部署上线** | 控制面 Docker 部署、节点接入手册、Nginx+HTTPS、备份/监控、压测与加固 | 商用上线 |

> 说明:同既往项目约定 —— 不一次性产出 200+ 文件的"能跑全栈",而是按上表逐阶段交付、每阶段实测。
> ⑤⑦ 的真实边缘能力需要你提供至少 1 台可作边缘的 VPS(OpenResty)与 1 个可托管 CNAME 的域名。

## 6. 启动前需你确认/提供

1. **品牌名与域名**:`AegisCDN` 是否沿用?CNAME 后缀域(如 `*.aegis-cdn.net`)用哪个真实域名?
2. **边缘服务器**:几台、地域、配置;是否叠加云清洗(决定"防护容量"真实上限)。
3. **支付**:Stripe 账号 / 是否需要国内支付(微信/支付宝)/ 对公。
4. **定价**:套餐最终价格(文档里是占位)。
5. **下一步**:确认后我从 **阶段①骨架 + ②官网首页** 开始落地。
