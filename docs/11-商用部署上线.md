# AegisCDN 商用部署上线 Runbook

面向真实商业运营的上线指南。架构分两类机器:**控制面集群**(官网/控制台/API/数据库)与
**边缘节点**(OpenResty 安全引擎,可多台、多地域)。

> 前置阅读:[09-安全引擎架构](09-安全引擎架构.md)、[10-本地全栈联调](10-本地全栈联调.md)。

---

## 1. 拓扑

```
                ┌── 控制面机器(1台起,可做主备/集群)──────────────┐
  用户/管理员 ──▶ │  Nginx(HTTPS)→ Next.js(:3000) + NestJS(:4000) │
                │  PostgreSQL · Redis · ClickHouse                  │
                └────────────▲────────────────────────┬─────────────┘
                    共用 Redis │ 下发配置        日志查询 │
   客户域名 ──CNAME──▶ ┌───────┴──── 边缘节点(N 台,多地域)─────────┐
  (xxx.com →          │  OpenResty 引擎(读 Redis 配置)+ edge-agent │
   节点IP / GeoDNS)   │  → 回源客户源站;日志 → 控制面 ClickHouse     │
                      └──────────────────────────────────────────────┘
```

**关键**:边缘节点必须能访问控制面的 **Redis**(读配置/封禁)与 **ClickHouse**(写日志)。
生产中这两个端口**只对边缘节点 IP 放行**(防火墙/安全组),不暴露公网。

## 2. 控制面部署(Docker)

### 2.1 准备
- 一台 2C4G+ 服务器(Ubuntu 22.04),装 Docker + compose。
- 两个域名:`console.example.com`(前端)、`api.example.com`(API)。
- 一个 CNAME 后缀域:`aegis-cdn.net`(分配给客户接入用),需你拥有并能配 DNS。

### 2.2 环境变量(生产)
`apps/api/.env`(参考 `.env.example`):
```
NODE_ENV=production
DATABASE_URL=postgresql://aegis:<强密码>@postgres:5432/aegis?schema=public
REDIS_URL=redis://redis:6379
CLICKHOUSE_URL=http://clickhouse:8123
JWT_SECRET=<openssl rand -base64 48>     # 必须 >=32 位,否则拒绝启动
CNAME_SUFFIX=aegis-cdn.net
CORS_ORIGINS=https://console.example.com
```
前端构建时:`NEXT_PUBLIC_API_URL=https://api.example.com/api/v1`

> 后端已内置:`NODE_ENV=production` 且 JWT_SECRET 为默认/弱值时**拒绝启动**(main.ts)。

### 2.3 起服务
- 用根 `docker-compose.yml` 起 postgres/redis/clickhouse/api(+ 如同机也可起 openresty/agent 做单机版)。
- 前端两种方式:① 容器化 Next standalone;② `npm run build:web && npm run start:web`(PM2 守护)。
- **生产建议**:内部端口(5432/6379/8123)**不映射到公网**,仅 compose 内网 + 边缘节点白名单访问。

### 2.4 HTTPS 反向代理
用 `deploy/nginx/control-plane.conf.example`(或宝塔)反代 :3000/:4000,certbot 签发证书。
前端 → `console.example.com`,API → `api.example.com`。

### 2.5 数据库 schema
首次:容器启动会 `prisma db push` 建表(见 compose api command)。
规范化迁移:在有 DB 时 `npx prisma migrate dev --name init` 生成迁移文件,之后用 `prisma migrate deploy`。

## 3. 边缘节点部署(每台)

1. 装 Docker;拉本仓库 `edge/` 目录。
2. 配置环境(指向控制面的 Redis/CK,用内网或专线/白名单公网):
   ```
   REDIS_HOST=<控制面内网IP>   REDIS_PORT=6379
   CLICKHOUSE_URL=http://<控制面内网IP>:8123
   AEGIS_SECRET=<与控制面一致的挑战密钥>
   ```
3. (可选)放入 `edge/openresty/geoip/GeoLite2-Country.mmdb` 启用地区拦截。
4. `docker compose -f docker-compose.edge.yml up -d --build`(或把 openresty+agent 跑成 systemd)。
5. **DNS 调度**:把客户分配的 CNAME(`<slug>.aegis-cdn.net`)解析到边缘节点 IP;多节点用 GeoDNS / 多 A 记录 + 健康检查摘除。

> 边缘与控制面共用 Redis 是闭环前提:控制面写 `aegis:cfg:*`,边缘读。务必同一 Redis(或主从同步)。

## 4. 监控与告警

- **健康检查**:`GET https://api.example.com/api/v1/health`(DB)、边缘 `GET /aegis-health`。接 UptimeRobot/Prometheus blackbox。
- **指标**:边缘节点 QPS/拦截量看 `/admin/security`(ClickHouse 聚合);系统层用 node_exporter + Grafana。
- **日志**:OpenResty `access.json.log`(已结构化)、NestJS stdout(docker logs / Loki)。
- **告警**:节点离线(agent 心跳停)、CK 写入失败、PG 连接数、磁盘水位、证书到期。

## 5. 备份

- `deploy/backup.sh`:每日 pg_dump + ClickHouse 导出,保留 14 天,**务必异地同步到对象存储**。
- crontab:`0 3 * * * /opt/aegis/deploy/backup.sh`。
- 演练恢复:定期用备份在测试环境 restore 验证可用。

## 6. 安全加固清单(上线前逐项确认)

- [ ] `JWT_SECRET` 为 48 位以上随机串;所有默认密码已改(PG/Redis/管理员账号)。
- [ ] PG/Redis/ClickHouse 端口**不暴露公网**,仅内网 + 边缘节点白名单。
- [ ] Redis 设密码(`requirepass`)+ 仅内网监听;ClickHouse 设访问账号。
- [ ] 控制面全站 HTTPS + HSTS;`CORS_ORIGINS` 仅真实前端域名。
- [ ] 攻击数据/管理接口已 `@Roles` 鉴权(已实现);首个注册者=平台管理员,注册后**关闭开放注册或限制**。
- [ ] 接入审核开启:客户域名需管理员审核才激活(已实现)。
- [ ] 速率限制:登录/注册接口加限流(反爆破);边缘 CC 阈值按业务调。
- [ ] 备份可用且异地;监控告警通畅。
- [ ] 合规:KYC 流程、违规处置流程、ToS/隐私页、备案(中国大陆)、敏感行业拒接策略。

## 7. 上线检查清单(Go-Live)

1. [ ] 控制面 `health` 200;前端可注册/登录;首个账号设为管理员。
2. [ ] 创建测试租户 → 加域名 → 管理员审核通过 → CNAME 生效 → 边缘真实代理。
3. [ ] 验证防护:SQLi→403、curl→挑战、洪水→限频、封禁 IP→即时拦截、封禁域名→403。
4. [ ] 计费:下单→开通→配额/功能门控生效(Starter 限 1 域名、无 CC/WAF)。
5. [ ] `/admin/security` 显示真实攻击数据(ClickHouse 实时)。
6. [ ] 备份任务跑通;监控告警到达;证书自动续期(certbot timer)。

## 8. 扩展与高可用(规模化时)

- 控制面 API 无状态可水平扩展(多副本 + 负载均衡);Redis 限流/缓存换集群。
- ClickHouse 单机→集群(分片+副本);日志量大时前置 Kafka 削峰(见 docs/09 §9)。
- 边缘节点按地域横向扩;DNS 智能调度 + 健康摘除;大流量 L3/L4 叠加上游云清洗。
- PostgreSQL 主从 + 只读副本;定期 vacuum/监控慢查询。

## 9. 仍需补齐(商用前/中)

- **真实支付**:当前 `payOrder` 是模拟。接 Stripe(国际)/微信支付/支付宝(国内)替换,配置 webhook 验签。

- 细粒度 RBAC、工单客服、发票、ASN 维度(GeoLite2-ASN)、JA3 完整解析。
